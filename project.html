Project guidelines

The goal of the final project is to present a small research project involving computational methods and tools, implemented using what you have learned during the course.

You may choose any subject or question, preferably one which is connected to your research or for which you have available data. If you have no available data or no idea for a research question, consult your supervisor or one of the course instructors.

Presentation

Projects will be presented as research posters at a poster session, which will take place on the afternoon of June 17th 2015.

Posters should contain the following sections:

Abstract
general background and overview of the project. What is the research question and what are the expected results?
Methods
description of the data used, the analysis performed, algorithm used to implement the analysis and statistical tests applied to the data.
Results
summary of the analysis results, statistical analyses etc.
Summary
conclusions drawn from the results, answers to the biological question and comparison to the expected results.
All sections may, and should, be accompanied by appropriate figures and tables, especially to clarify the results and their biological meaning.

Ask your adviser for the required funding. If this is a problem, consult one of the instructors.

Code

You should use python code to perform the processing and analysis of the data, . You may use any python module you see fit or any external code, as long as the main body of the code is original.

External software and tools such as Microsoft Excel, text editors or analytic software may be used in processing the data, and if possible embedded within the python code. The core analysis should be performed with python code, including statistical analysis, plotting etc. If you are not sure, please ask in the forum or via email.

Project submission

The final submission will include the following:

Poster
as presented in the poster session. A single PDF file.
Code
all the code used for the processing and analysis of the data should be included in one IPython notebook. The code should be readable and clear, including comments, function documentation, explanations and usage instructions outside the code. It is important that the code can be run and the results are reproducible.
Data
the original data on which analyses were performed. Again, make sure that using the code on the provided data, the results may be reproduced. The data can either be added to the submission in a zip file or a link should be provided inside the notebook.

Abstract

An abstract must be submitted before May 1st. Abstracts should be submitted as one page PDF documents, briefly depicting the biological background, the data and the planned analyses. Include the names on other researchers that will help you with the data collection or analysis.

Grading

Project grading will be based on research problem (shouldn't be trivial), presentation quality, code readability and clarity, code efficiency (in terms of what we learned in the course), usage of the right tools and modules.

Help

Questions regarding the project are welcome. Use the course forum or consult one of the instructors for questions about the instructions or the programming involved. You are also encouraged to consult your adviser or anyone else; however, the actual code should be written by you. Please start working on the projects as soon as possible and ask your questions early, so we can attend to all issues.
